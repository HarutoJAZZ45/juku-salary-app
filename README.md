# Juku Salary App 💰

塾講師の給与管理を劇的に効率化する、オフラインファーストのWebアプリケーションです。
日々の複雑なコマ給計算、事務給、交通費、ヘルプ手当などを自動集計し、リアルタイムで給与見込みを可視化します。

---

## 🛠 技術スタック

- **Frontend**: [React 18](https://reactjs.org/) + [TypeScript](https://www.typescriptlang.org/)
- **Build Tool**: [Vite](https://vitejs.dev/)
- **Backend / Database**: [Firebase](https://firebase.google.com/)
    - **Authentication**: Google/Email 認証
    - **Firestore**: リアルタイム同期・クラウドバックアップ・グローバルランキング
- **Icons**: [Lucide React](https://lucide.dev/)
- **Styling**: Vanilla CSS (CSS Modules的なアプローチを意識した設計)

---

## 🏗 システムアーキテクチャ

### 1. オフラインファースト設計
本アプリはユーザー体験を損なわないよう「オフラインでも遜色なく使える」ことを重視して設計されています。

- **LocalStorage**: 全ての勤務データと設定は、即座にブラウザの LocalStorage に保存されます。ネットワークの状態に関わらず瞬時に反応します。
- **Firebase 同期 (`useSalaryData`フック)**: ログイン中、データ変更（`entries` や `settings`）を検知し、バックグラウンドで Firestore へ差分同期を行います。
- **データ衝突の解決**: ログイン時にはクラウドのデータをマスターとして扱い、ローカルデータを上書きします。一方、未ログイン状態で入力したデータは、ログイン後の初回同期時にクラウドへ自動アップロードされるシームレスな移行フローを実装しています。

### 2. 状態管理の設計
- **Domain Logicのカプセル化**: データ操作 (`useSalaryData`) や認証ロジック (`useAuth`) はカスタムフックに集約され、UI コンポーネントからは純粋な関数呼び出しとして扱われます。
- **Context API によるグローバル設定**: 言語設定 (i18n) やテーマに関わる状態は Context API で管理され、Prop Drilling を防いでいます。

## 🔢 実装の詳細（コアロジック）

### 1. 給与計算アルゴリズム (`src/utils/calculator.ts`)
塾講師特有の計算ルールを、堅牢な純粋関数として実装しています。

- **授業給（コマ給）**: 
    - 1コマ(90分)を「1.5時間 × 時給」として算出。
    - **役職給**: リーダー（2000円ベース）やサブリーダー（1500円ベース）が設定されているコマは、個別の割増レートで優先計算されます。
- **ヘルプ勤務調整**:
    - 設定された「所属校舎」以外での勤務を検知すると、契約に基づいた時給の加減算（例：平岡校舎以外へのヘルプ時は時給-100円など）を動的に適用します。
- **事務給（生徒対応給）の自動精密計算**: 
    - 手入力の業務時間に加え、**「各コマ5分の準備時間」**と、**「連続するコマ間に発生する10分休憩（授業間）」**を自動集計。
    - ただし、BコマとCコマの間（昼休憩時間帯）は給与対象外として除外するロジックを組み込んでいます。
- **勤務手当**:
    - 勤務校舎の地域属性（平岡地区か否か）を判別し、一律の勤務手当を自動加算します。

### 2. 称号・レベルシステム (`src/utils/levelSystem.ts`)
ゲーミフィケーション要素を裏側で支えるXP計算エンジンです。

- **成長ロジック**: 累積の授業数、出勤日数、獲得したバッジの難易度に基づき、独自の計算式で XP を算出。
- **動的アンロック**: `useEffect` による監視と組み合わせ、条件を満たした瞬間に新しい「称号（Title）」をプロフィール設定項目へ解禁します。

## 📂 ディレクトリ構造

```bash
src/
├── components/   # UIコンポーネント（Modal, Card, 共通パーツ）
├── contexts/     # グローバルContext（言語設定、UI状態）
├── hooks/        # 核心ロジック（給与データ管理、認証、お知らせ、バッジ同期）
├── lib/          # Firebase SDK 初期化設定
├── locales/      # 多言語対応用辞書ファイル (ja, en, es)
├── utils/        # 純粋な計算ロジック、フォーマット関数、集計
├── data/         # お知らせデータ、静的なマスターデータ
└── types/        # TypeScript 型定義 (Interface / Type)
```

## 🌍 国際化 (i18n)
`locales/` 配下の辞書ファイルと言語コンテキストを組み合わせ、レンダリング時に動的に解決します。これにより、単一のコードベースで日本語・英語・スペイン語を完全にサポートしています。
